# `_archive/` — Internal Working Notes

This folder contains private development history and planning documents that are kept in version control but are **not part of the public-facing project documentation**.

Files here are intentionally not surfaced in the repo root, the README, or the npm package. They exist so that the project's history and decision trail remain recoverable.

## Layout

- `sessions/` — chronological session save files documenting what was built when, and what the next-session priorities were
- `architecture/` — superseded architectural plans, upgrade roadmaps, and design checklists (kept for traceability of decisions)
- `launch-drafts/` — draft posts for Hacker News, LinkedIn, X, etc. (kept so I can reuse phrasing and remember what worked)

## Convention

When a planning doc is superseded by a newer version, the old one moves here rather than being deleted. When a launch happens, the draft moves here rather than being deleted.

The repo root stays clean. The history stays intact.
